Developer FAQ

Common issues and solutions you could encounter while using the Legends SDK

- What is the SideQuest Inspector and what benefit does it offer when using the Legends SDK?
The SideQuest Inspector is the most powerful tool in the Legends SDK. The beautiful part about the SideQuest Inspector is that it allows you to run the full Legends game platform inside the Unity3D editor with full access to the Unity3D tools for inspecting, debugging and improving performance. This means we can allow developers to have as much power as we do to build the best apps they can on Legends.


- Do I need the SideQuest Inspector?
When we started building the Legends SDK we realised that to provide developers with powerful scripting capability was pretty awesome but pretty soon it was clear we had to do more in order to provide a complete developer solution. We needed to find a way to give developers as much power as us when we build Legends. To solve this problem we built the SideQuest Inspector mode into our SDK. The inspector lets you run the entire legends platform inside unity giving you ultimate control to be able to leverage unity to help debug your applications.


- What can't I do with the Legends SDK?
The legends SDK is designed to give developers as much control and power as possible to be able to build truly interactive and functional applications. We have disabled some features to protect users and their data and there are also some other unrelated gotchas too.

- You cannot access the System.IO namespace.
- You cannot use reflection.
- Some other namespaces are also disabled.
- Until approved by us your custom scripts cannot run on other user's clients unless they agree.
- Public spaces will need to be approved before they will function on everyone's client.
- You cannot use enums ( Mono Cecil limitation ).
