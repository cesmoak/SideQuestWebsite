Getting Started - Build an App

Back To Developers
Get Unity3D Setup to use the SideQuest Legends SDK

Step 1 - Open
Open the SideQuest option in the Tools menu of Unity3D as seen in the image above.


Step 2 - Select a GameObject
Select a GameObject in your hierarchy panel. If you see any errors or warnings at this point see here.


Step 3 - Script Scan
Click Next to scan the selected game object for AppBehaviours or custom scripts.


Step 4 - Select Scripts & Extra Assets
Note: The defaults here are fine in most cases. If you are unsure just click Next.

Scripts
This will scan the selected GameObject for an AppBehaviour script and attempt to find any dependencies for it automatically. The pre-selected items here have been detected as dependencies, but you may also need to select more scripts if their types are not defined in the AppBehaviour anywhere.

Assets
Assets that are referenced inside the selected GameObject or any of its children are automatically added to the output App - selecting them again here will have no effect. This screen is intended for adding any extra assets that you wish to access from a script but are also not referenced in the selected GameObject anywhere.

Click Next move to the Build Screen


Step 5 - Quick Build Option
Select Quick Build to skip the mobile build variant - this greatly reduces build times during testing and allows you to test the app on desktop ( or mobile with Oculus Link ). You only need to do a complete build when you are ready to publish your app.


Step 6 - Run Build
Select Build SideQuest App to start the build process. Keep an eye on the console tab for any issues. See here for an FAQ of common errors.


Grab your squanch files!
Your app will be compiled into the output folder. If you selected quick build there will be one file otherwise there will be two, gameobject.squanch and gameobject.squanch_android. These files can be loaded directly into SideQuest Legends by creating a space here or you can run them in the SideQuest Inspector to test.
